    저장소 받기 (Git clone)¶

    왜 필요한가요?

    MkDocs 문서만 보면 개념은 따라갈 수 있지만, Terraform 실습은 노트북에 .tf 파일이 있어야 terraform init / plan / apply를 할 수 있습니다. 강사가 안내한 Git 저장소를 clone해 두세요.

    빠른 시작 (대부분의 수강생)¶

    아래 명령어를 터미널에 그대로 복사-붙여넣기 하세요.

    cd ~
mkdir -p git && cd git
git clone https://github.com/noisonnoiton/az-104-labs.git
cd az-104-labs

    강사가 다른 URL을 안내한 경우

    위 URL 대신 강사가 알려준 URL로 바꿔 실행합니다.

    clone이 끝나면 Terraform Lab 폴더로 이동합니다.

    cd labs/terraform
ls
# 00-skeleton/  lab04-virtual-networking/  ...

    clone 완료

    • 첫 실습: 00 - Skeleton (RG verify)
    • Terraform Lab 안내: labs/terraform/README.md

    아래는 참고 사항입니다. 위 "빠른 시작"으로 진행이 되면 아래 내용은 건너뛰어도 됩니다.

    선행: Git 설치 확인¶

    Windows, macOS 설치 문서에서 Git까지 끝냈는지 확인합니다. Windows | macOS

    git --version

    저장소 URL 형태¶

    방식 예시 형태 비고
    HTTPS https://github.com/<조직>/<저장소>.git 처음 쓰기 쉬움
    SSH git@github.com:<조직>/<저장소>.git SSH 키를 등록한 경우

    문서만 볼 때

    읽기 전용 문서는 GitHub Pages 등으로 열 수 있습니다(강사 안내 URL). Terraform 실습까지 하려면 반드시 코드가 포함된 저장소를 clone하세요.

    Private 저장소인 경우¶

    • GitHub: 브라우저 로그인 후 Personal Access Token(PAT) 또는 Git Credential Manager로 HTTPS 인증하는 경우가 많습니다. 세부 단계는 GitHub 문서를 참고하거나 강사에게 물어보세요.
    • 조직 정책상 SSH만 허용이면, SSH 키 등록 후 git@github.com:... URL로 clone합니다.

    Git 없이 받는 방법 (비권장)¶

    • GitHub 웹에서 Code → Download ZIP으로 내려받을 수는 있습니다.
    • 이후 git pull으로 업데이트가 어렵고, 실습 중 버전이 바뀌면 다시 받아야 하므로 가능하면 git clone을 권장합니다.
