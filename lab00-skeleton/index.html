    Lab 00 - Terraform Skeleton (RG Verify)¶

    이 skeleton에서 하는 일

    할당된 Resource Group에 연결할 수 있는지, 권한이 올바른지 검증합니다. 다른 Lab을 시작하기 전에 가장 먼저 한 번 실행하세요.

    다음 단계

    검증이 끝나면 Lab 04 - Virtual Networking으로 이동합니다.

    Architecture Diagram¶

    Terraform data source로 기존 RG를 읽기만 하며, 새 리소스를 생성하지 않습니다.

    Assumptions¶

    • 강사가 과정용 subscription 안에 수강생별 Resource Group을 만들고, 해당 RG에 Contributor(또는 동급) 권한을 부여한 상태입니다.
    • 수강생은 subscription Owner가 아닐 수 있습니다. Terraform은 할당된 RG 안에서만 동작합니다.

    반드시 확인

    terraform.tfvars의 resource_group_name이 강사가 알려준 본인 RG 이름과 일치하는지 확인하세요. 다른 사람의 RG나 다른 subscription에 적용하지 마세요.

    Prerequisite¶

    항목 설명
    Azure CLI 설치 후 az version으로 확인
    Terraform 설치 1.6+, terraform version으로 확인
    인증 가이드 Azure 접근 및 Terraform 인증
    OS 설정 Windows 또는 macOS

    Step-by-Step¶

    Terraform 코드 위치: labs/terraform/lab00-skeleton/

    1) Prepare variables¶

    예제 파일을 복사하고 본인 값을 입력합니다.

    cd labs/terraform/lab00-skeleton
cp terraform.tfvars.example terraform.tfvars

    에디터로 terraform.tfvars를 열고 다음을 설정합니다.

    • resource_group_name (필수) — 강사가 알려준 본인 RG 이름
    • subscription_id (권장) — 강사가 안내한 subscription GUID

    2) Login to Azure¶

    az login

    올바른 subscription이 선택되었는지 확인합니다.

    az account show -o table

    3) Initialize Terraform¶

    필요한 provider 플러그인을 다운로드합니다.

    terraform init
    예상 출력 
    Initializing the backend...
Initializing provider plugins...
- Finding hashicorp/azurerm versions matching "~> 4.0"...
- Installing hashicorp/azurerm v4.x.x...
Terraform has been successfully initialized!

    4) Plan¶

    Terraform이 무엇을 할지 미리 확인합니다. 이 skeleton에서는 기존 RG를 읽기만 합니다.

    terraform plan
    예상 출력 
    data.azurerm_resource_group.main: Reading...
data.azurerm_resource_group.main: Read complete

No changes. Your infrastructure matches the configuration.

    Resource Group "xxx" was not found 에러가 나오면 resource_group_name과 subscription을 다시 확인하세요.

    5) Apply¶

    apply를 실행하여 Terraform이 RG에 연결할 수 있고, 권한이 올바른지 최종 검증합니다.

    terraform apply

    프롬프트에 yes를 입력합니다. 정상 출력이 나오면 RG 연결과 권한이 확인된 것입니다.

    예상 출력 
    Apply complete! Resources: 0 added, 0 changed, 0 destroyed.

Outputs:
resource_group_id   = "/subscriptions/.../resourceGroups/rg-az104-student01"
resource_group_name = "rg-az104-student01"

    6) Cleanup¶

    이 skeleton은 과금 리소스를 생성하지 않지만, state를 정리합니다.

    terraform destroy

    프롬프트에 yes를 입력합니다.

    다음 단계

    이 skeleton은 RG 연결 검증만 합니다. 실제 리소스 배포는 Lab 04 - Virtual Networking 등 별도 Lab에서 진행합니다.

    Terraform Files¶

    파일 역할
    versions.tf Terraform / provider 버전 제약
    providers.tf azurerm provider 설정
    variables.tf 입력 변수: RG 이름, 지역, 태그
    main.tf data.azurerm_resource_group + 출력 (연결 검증)
    terraform.tfvars.example 변수 예시 파일 (비밀 값 없음)

    Troubleshooting¶

    증상 원인 해결
    Resource Group "xxx" was not found RG 이름 오류 또는 다른 subscription terraform.tfvars와 az account show 확인
    Error: building AzureRM Client 로그인 안 됨 또는 토큰 만료 az login 재실행
    terraform init provider 에러 네트워크 문제 또는 프록시 인터넷 연결 확인 후 terraform init 재시도

    Cost & Caution¶

    실수 방지

    terraform.tfvars의 resource_group_name이 반드시 본인 RG인지 확인한 뒤 apply하세요. 잘못된 subscription이나 RG에 적용하면 다른 수강생의 리소스에 영향을 줄 수 있습니다.
