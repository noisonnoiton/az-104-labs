    Custom Lab Cleanup¶

    이 Lab은 별도 인프라를 생성하지 않습니다

    Lab 90 · 91 · 92에서 만든 리소스를 역순으로 삭제하는 정리 절차입니다.

    사전 조건¶

    항목 설명
    Lab 90 · 91 · 92 완료 세 Lab을 모두 마친 뒤 진행
    Terraform Lab 90 · 92의 Terraform state가 있는 환경
    Azure CLI 로그인 az login 완료

    삭제 순서 개요¶

    AKS 클러스터를 terraform destroy하면 그 위에 배포된 모든 Kubernetes 리소스(앱, 인프라, OTel Collector 등)가 함께 삭제됩니다.
    따라서 kubectl delete는 불필요하며, Terraform destroy 2회만 실행하면 됩니다.

    Lab 92 (Azure Monitor)  →  Lab 90 (AKS 클러스터)
   terraform destroy           terraform destroy
    단계 대상 삭제되는 리소스
    1 Azure Monitor (Lab 92) Log Analytics Workspace, Application Insights
    2 자동 생성 리소스 Smart Detection Action Group, Failure Anomalies Alert Rule, Data Collection Rule
    3 AKS 클러스터 (Lab 90) AKS, Node Pool, VNet, ACR Role Assignment + 클러스터 내 모든 K8s 리소스

    왜 kubectl delete가 필요 없나요?

    AKS 클러스터를 삭제하면 클러스터 내 모든 namespace, Pod, Service, Ingress, PVC 등이 함께 제거됩니다.
    Lab 91에서 배포한 앱(awesome), 인프라(awesome-infra), Lab 92의 OTel Collector 모두 포함됩니다.

    Step-by-Step¶

    1) Azure Monitor 리소스 제거 (Lab 92)¶

    cd labs/terraform/lab92-aks-monitoring
terraform destroy

    프롬프트에 yes를 입력합니다.

    삭제되는 리소스
    • Log Analytics Workspace (law-az104-lab92)
    • Application Insights (appi-az104-lab92)

    2) 자동 생성 리소스 수동 제거¶

    Application Insights와 Container Insights가 자동 생성한 리소스는 Terraform이 관리하지 않으므로 Portal에서 수동 삭제합니다.

    Azure Portal → Resource Groups → 본인 RG에서 아래 리소스를 선택하고 Delete:

    리소스 이름 유형
    Application Insights Smart Detection Action group
    Failure Anomalies - appi-az104-lab92 Smart detector alert rule
    MSCI-koreacentral-aks-az104-aks Data collection rule

    자동 생성 리소스란?

    • Failure Anomalies / Smart Detection: Application Insights 생성 시 자동 생성되는 Alert Rule + Action Group
    • MSCI-...: az aks enable-addons --addons monitoring 실행 시 자동 생성되는 Data Collection Rule

    3) AKS 클러스터 제거 (Lab 90)¶

    cd ../lab90-aks
terraform destroy

    프롬프트에 yes를 입력합니다. AKS 삭제에 수 분 소요됩니다.

    삭제되는 리소스
    • AKS 클러스터 (aks-az104-lab90)
    • User Node Pool
    • VNet / Subnet
    • ACR Role Assignment (AcrPull)
    • MC_ 리소스 그룹 (AKS가 자동 생성한 노드 RG)
    • 클러스터 내 모든 K8s 리소스 (앱, 인프라, OTel Collector, PVC 등)

    삭제 확인¶

    모든 리소스가 정리되었는지 확인합니다.

    # Terraform state 확인 — 비어 있어야 함
cd labs/terraform/lab90-aks && terraform state list
cd ../lab92-aks-monitoring && terraform state list

# Azure 리소스 확인
az resource list -g <본인 RG 이름> -o table

    비용 방지

    AKS 노드 VM(Standard_D2s_v3 × 2)은 시간당 과금됩니다. terraform destroy가 정상 완료되었는지 반드시 확인하세요.
    Azure Portal → Resource Groups → 본인 RG에서 AKS 관련 리소스가 남아 있지 않은지 시각적으로도 확인하는 것을 권장합니다.
