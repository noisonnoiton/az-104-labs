1. 🧭 What is Kubernetes?
Kubernetes (pronounced koo-ber-neh-tees) is an open-source platform that automatically manages, deploys, and scales your applications packaged inside containers.

The name comes from the Greek word for “helmsman” or “pilot” — the person who steers a ship.

It:
Runs apps in containers
Keeps them alive
Scales them when needed
Fixes failures automatically

👉 Kubernetes = the system that steers your applications safely

4. 🏗️ The Kubernetes Architecture
A Kubernetes system is called a Cluster

Press enter or click to view image in full size

A Kubernetes environment consists of two main components:
Cluster = Control Plane + Worker Nodes
☁️ Control Plane 🧠
Manages the cluster : Key components include

| ------------------ | ----------------------- |
| Component          | Function                |
| ------------------ | ----------------------- |
| API Server         | Entry point             |
| etcd               | Cluster database        |
| Scheduler          | Assigns pods            |
| Controller Manager | Maintains desired state |
| Cloud Controller   | Cloud integrations      |
| ------------------ | ----------------------- |
⚙️ Worker Nodes 💪
Worker nodes run application containers.

| ----------------- | ------------------- |
| Component         | Function            |
| ----------------- | ------------------- |
| kubelet           | Runs containers     |
| kube-proxy        | Networking          |
| Container Runtime | Executes containers |
| ----------------- | ------------------- |
These nodes execute workloads in the form of Pods.

📦 Pods
👉 Pods are the smallest deployable unit in Kubernetes.

🔑 Key Characteristics:
Smallest deployable unit
Usually contains one container (most common)
Can contain multiple tightly coupled containers
Runs on a worker node
Gets its own IP address
Ephemeral (temporary)
Even though your applications run inside containers (like Docker), Kubernetes never deploys containers directly. Instead, it wraps them inside Pods, which act as the fundamental building blocks of your cluster.

🧠 Pods are the heart of Kubernetes architecture.

⚙️ How Pods Work (Simple View)
Press enter or click to view image in full size

Inside a Pod:

Containers share:
🧠 Network (same IP, communicate via localhost)
💾 Storage (shared volumes)
🔄 Lifecycle (start/stop together)
👉 This makes Pods perfect for tightly coupled components.

📦 Pod & Container Relationship
✅ 1 Pod = 1 Container (Most Common)
Typical for web apps (NGINX, Node.js, Python apps)
Easy to scale and manage
⚠️ 1 Pod = Multiple Containers (Advanced Use Case)
Used when containers must work together closely, for example:

App container + log processor
App container + sidecar proxy
App container + file processor
👉 These are called Sidecar Patterns

🔄 What Is Container Orchestration?
When applications run multiple containers, managing them manually becomes difficult.

Kubernetes solves this by:

Automatically scheduling containers
Managing failures and recovery
Scaling based on demand
Handling networking between services
Kubernetes is the most widely used container orchestration platform today.

⚔️ Docker vs ContainerD (Simple Difference)
Docker → Full platform for building and running containers
ContainerD → Lightweight runtime used internally by Kubernetes
👉 Kubernetes now mainly uses ContainerD for better performance

Docker vs ContainerD
Both Docker and ContainerD are container runtimes.

| ------------------------ | ------------------ | ----------------- |
| Feature                  | Docker             | ContainerD        |
| ------------------------ | ------------------ | ----------------- |
| Type                     | Container platform | Container runtime |
| Includes CLI tools       | Yes                | No                |
| Kubernetes compatibility | Deprecated runtime | Native support    |
| Lightweight              | Medium             | Very lightweight  |
| ------------------------ | ------------------ | ----------------- |
Modern Kubernetes clusters use ContainerD as the default runtime.

16. 📈 Scaling in Kubernetes (Very Important)
Press enter or click to view image in full size

When traffic increases:

❌ You DO NOT add containers to the same Pod
✅ You create new Pods

Example:
1 Pod → 1 App instance
3 Pods → 3 App instances
10 Pods → High scalability
👉 This is called Horizontal Scaling

🧩 Pods Across Nodes
If one node is full → Kubernetes schedules Pods on another node
Cluster automatically expands capacity
Pods are distributed across nodes
